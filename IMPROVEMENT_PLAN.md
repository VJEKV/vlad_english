## КОРНЕВЫЕ ПРИЧИНЫ ПРОБЛЕМ С ОЗВУЧКОЙ

**Проблема:** OpenAI TTS генерирует mp3 → возвращает base64 → Audio элемент воспроизводит. Но:
- Base64 строка для предложения = 50-200KB текста через IPC — может тормозить
- OpenAI API может не отвечать или отвечать медленно
- Fallback на Web Speech API звучит как робот
- Нет индикации какой TTS работает — пользователь не знает что происходит

**Решение: Стратегия PRE-GENERATION + CACHE**

При первом открытии модуля Spotlight — предгенерировать ВСЕ аудио для слов и предложений в фоне:
1. Запуск модуля → показать "Готовлю озвучку..." (1-2 мин один раз)
2. Сгенерировать mp3 для каждого слова (500 слов × ~$0.001 = $0.50)
3. Сохранить в кеш на диск
4. При воспроизведении — играть из кеша (мгновенно, офлайн)
5. Для слогов — нарезать аудио слова на части по таймингам

### Этап 3: Чтение по слогам (правильная реализация)

**3.1 Предгенерировать слоги:**
- При первом чтении слова — сгенерировать mp3 для каждого слога

- Кешировать навсегда

**3.2 Анимация:**

**3.3 В предложениях (KaraokeText):**

## ТЕХНИЧЕСКИЕ РЕШЕНИЯ

### Озвучка — новая архитектура

```
Слой 1: Кеш (мгновенно, офлайн)
  └→ tts-cache/[hash].mp3 на диске

Слой 2: OpenAI TTS (качество 9/10, нужен ключ + интернет)
  └→ gpt-4o-mini-tts, голос "nova"
  └→ Генерирует mp3 → сохраняет в кеш → возвращает base64

Слой 3: Edge-TTS (качество 7/10, бесплатно, нужен интернет)
  └→ en-US-JennyNeural / ru-RU-SvetlanaNeural
  └→ Fallback если нет OpenAI ключа

Слой 4: Web Speech API (качество 3/10, офлайн)
  └→ Последний fallback, только если всё остальное не работает
```

### Предгенерация слов модуля

```typescript
// При входе в Spotlight модуль:
async function preGenerateModuleAudio(words: string[]) {
  for (const word of words) {
    // 1. Целое слово
    await window.electronAPI.tts.speak(word, 'en', '+0%');
    // 2. Слоги
    for (const syllable of getSyllables(word)) {
      await window.electronAPI.tts.speak(syllable, 'en', '-20%');
    }
  }
}
```
